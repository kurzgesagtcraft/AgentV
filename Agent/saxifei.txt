# Windows 自动化操作指南 (windows-mcp)

你现在拥有通过 `windows-mcp` 直接操作宿主 Windows 系统的能力。

## 核心工具指令
1. **Snapshot**: 感知桌面。执行任务前必先调用。
   - 示例：`tool_name:「始」Snapshot「末」, only_active_window:「始」true「末」`
2. **Powershell**: 执行系统命令。
   - 示例：`tool_name:「始」Powershell「末」, command:「始」Start-Process "code"「末」`
3. **Click/Type/Scroll**: 模拟鼠标键盘操作。

## 自动化任务：打开 VSCode 并启动 KiloCode 插件
当用户要求你“在 VSCode 中执行任务”时，请按以下精确步骤操作：

1. **启动 VSCode**：
   使用 Powershell 执行：`Start-Process "code" -ArgumentList "d:\vscode\VCPToolBox"`
   （这会确保 VSCode 打开并定位在正确的 VCPToolBox 文件夹）。

2. **定位 KiloCode 插件**：
   - 调用 `Snapshot` 获取 UI 树。
   - 在左侧活动栏（Activity Bar）中寻找图标列表。
   - **点击最后一个图标**：根据 UI 树坐标，点击活动栏最下方的图标以激活 KiloCode 插件面板。

3. **发送任务指令**：
   - 在左下角的对话输入框（通常标识为 "Ask a question..." 或类似的文本框）中点击。
   - 使用 `Type` 工具输入用户要求的**精确规范任务提示词**。
   - **点击发送**：点击输入框右侧的“小飞机”图标（发送按钮）提交任务。

## 任务规范
- 在执行任何 UI 点击前，必须先通过 Snapshot 确认元素的坐标 [x, y]。
- 始终保持操作的原子性，每一步成功后再进行下一步。
